# BAEKJOON ONLINE JUDGE - 1933. 스카이라인

- [문제출처](https://www.acmicpc.net/problem/1933 '1933. 스카이라인')

## 알고리즘 분류

- 자료 구조
- 트리를 사용한 집합과 맵
- 스위핑
- 우선순위 큐

## 풀이

### 접근

- 우선순위 큐

### 설계

- 주어진 좌표들을 최소 힙에 저장
- 오른쪽 `x` 좌표를 `end`로 설정하고 왼쪽 `x` 좌표와 `end`를 비교하여 `stack`에 저장
  - 왼쪽 `x` 좌표가 `end` 보다 크다면 새로운 건물이므로 이전 건물의 마지막 좌표와 높이 `0`을 `stack`에 저장하고 새로운 건물의 좌표를 저장
  - 왼쪽 `x` 좌표가 `end`와 같고, `stack`의 마지막 높이와 다르다면 마찬가지로 `stack`에 왼쪽 `x` 좌표와 높이를 저장
    - 두 과정을 진행하고 `end`를 오른쪽 `x` 좌표로 업데이트
  - 왼쪽 `x` 좌표가 `end` 보다 작다면 건물이 겹치는 경우이므로 현재 높이와 이전 높이를 비교
    - 현재 높이가 더 작다면 `stack`은 갱신하지 않고, 오른쪽 `x` 좌표가 `end`보다 크면 힙에 `end`와 높이, 오른쪽 `x` 좌표를 저장
    - 현재 높이가 더 크다면 `stack`을 갱신하고, `end`가 오른쪽 `x` 좌표보다 크면 힙에 오른쪽 `x` 좌표, 이전 높이, `end`를 저장
