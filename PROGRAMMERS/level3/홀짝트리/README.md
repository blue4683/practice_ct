# Programmers - Level 3. 홀짝트리

- [문제출처](https://school.programmers.co.kr/learn/courses/30/lessons/388354 'Level 3. 홀짝트리')

## 풀이

### 접근

- `유니온 파인드`

### 설계

- 주어진 edges로 유니온 파인드를 수행하고 각 노드의 자식 개수를 업데이트
- 각 노드의 홀수/짝수과 노드의 자식 개수의 홀수/짝수를 비교하고 이것을 유니온 파인드로 수행했을 때 parent 노드를 인덱스로 갱신해줌
  - parent 노드를 기준으로 홀수/짝수 또는 역홀수/역짝수 노드의 개수가 1개라면 그 노드를 루트로 했을 때 루트를 제외한 나머지 루트의 자식 개수가 1개씩 줄어들기 때문에 결과적으로 모든 노드가 루트 노드와 같아짐
- parent 대표 노드들을 탐색하며 특징이 다른 노드가 딱 1개 있을 때 그에 해당하는 트리의 개수를 갱신해줌
